## Phase 1: Core Observation UX (最優先: 顕微鏡のピント合わせ)
目的：とにかく「見やすく」し、データへの気づきを増やす。
**「軽さを最優先」「ノイズを排除」「思考を止めない」** 状態を目指す。

### P0 (Must Have)（完了）
- [x] 1-1. NULL / 空文字 / 0 の視覚的な区別 (ノイズにならない範囲の最小限の色付けや記号)
  > 空文字を `""` 表示に変換。NULL は `NULL` 文字列のまま。ScanRows 内で分岐。
- [x] 1-2. 型情報表示 (ヘッダ等にカラムの型をさりげなく示す)
  > `rows.ColumnTypes()` → `DatabaseTypeName()` でヘッダに `name text` 形式で表示。
- [x] 1-3. ソート（単キー: SQLを書き直さずにヘッダ選択で並び替え。思考を止めない）
  > h/l でカラム選択、s でトグル (None→Asc→Desc)。smartCompare で数値/文字列判定、NULL は常に末尾。sort.go に分離。
- [x] 1-4. ページング機能と状態の可視化 (PgUp/PgDn高速移動、ページサイズ変更、`101-200 / total` 現在位置表示)
  > ステータスバーに `col:name 1/100` 形式の位置表示を追加。PgUp/PgDn は bubbles table のデフォルトキーマップで対応済み。
- [x] 1-5. クエリ履歴 (過去に実行したクエリの呼び出し。入力のストレスを減らす)
  > セッション内履歴 (最大100件)。INSERT モードで Ctrl+P/Ctrl+N でナビゲーション。

### P1 (Should Have)
- [x] 1-6. 折り返し表示 or 行詳細表示 (セル内の長いテキストを確認する手段)
  > Detail View Mode として実装。Enter でオーバーレイ表示、j/k でフィールド移動、n/N で行遷移、q/Esc/Enter で閉じる。型名短縮表示 (INTEGER→int 等) も同時に実装。
- [x] 1-7. 保存クエリ (スニペット機能)
  > `~/.config/asql/snippets.yaml` に永続化。NORMALで `S` 一覧 / `Ctrl+S` 保存、INSERTで `Ctrl+S` 保存。enterSnippetNamingMode ヘルパーで重複排除。
- [ ] 1-8. 接続プロファイル管理の永続化 (複数DBの認証情報を設定ファイル等に保存。#16 / 0-3 と統合)
- [ ] 1-9. テーブル名・カラム名の入力補完 (Tabキー。TUI内で思考を止めない)
- [ ] 1-10. クエリ履歴のインクリメンタル検索 (Ctrl+R等。TUI内で過去の試行を即座に再利用)
- [ ] 1-11. TUIエディタ操作の洗練 (行削除やカーソル移動などのショートカット強化)
- [x] 1-12. PgUp/PgDn キーによるテーブル高速スクロール (bubbles table デフォルトで効かない場合の対応)
  > `updateNormal` の switch に `tea.KeyPgUp, tea.KeyPgDown` case を追加し `m.table.Update(msg)` に委譲。
- [ ] 1-13. 多カラムテーブルの表示崩れ修正 (列数が画面幅を超える場合にスキーマ表示が折り返し、データ行が描画崩れする問題。水平スクロール対応 or 表示列の動的制限が必要)

## Phase 2: Multi-DB Observation (比較の自動化)
目的：**「観察を加速する」**。本番と検証、あるいは異種DB間での「差」を浮き彫りにする。

- [ ] 2-1. 複数接続同時保持 (セッション内で複数DBのコネクションを持つ)
- [ ] 2-2. 同一クエリを別DBで実行 (SQLを書き直さずに接続先だけ切り替えて再実行)
- [ ] 2-3. 横並び表示 (2つの結果セットを画面分割等で並べて比較)
- [ ] 2-4. 件数差表示 (差分ハイライト等で違和感に即座に気づかせる)

## Phase 3: Bring & Join (ローカル集約分析)
目的：**「Bring Data Philosophy」**の体現。異種DBを直接統合せず、ローカルに持ち寄って気づく。
前提：Phase 2 の複数接続基盤。

- [ ] 3-1. クエリ結果をローカル一時テーブルに保存 (SQLite等)
- [ ] 3-2. ローカルでのJOIN実行 (複雑さよりも制御可能性を優先)
- [ ] 3-3. 日次などの粒度統一サポート

## Phase 4: Light Insight Helpers (軽量補助)
目的：**「軽さを損なわない範囲で、気づきを増やす」**。
※ Phase 2/3 に依存しない。Phase 1 完了後、優先度に応じて着手可能。

- [ ] 4-1. NULL率表示 (軽量情報への即時アクセス)
- [ ] 4-2. distinct数表示
- [ ] 4-3. min/max表示
- [ ] 4-4. 件数推移の簡易表示 (スパークライン等。目立たないUIで傾向を示す)
- [ ] 4-5. 簡易ヒストグラム表示 (将来)
