# asql Project Mandates

このプロジェクトにおける Gemini CLI の行動規範。`VISION.md` に基づく。

## Core Principles (行動原則)
実装の提案やコード変更を行う際、常に以下の原則を最優先すること。

1. **軽さを最優先する**:
   - 起動速度、反応速度を損なう変更を避ける。
   - 不要な描画や重い外部依存の追加に慎重になる。
2. **思考を止めない**:
   - キーボード操作の効率性を重視する。
   - ユーザーが SQL を書き直さずともデータを探索できる TUI UX を追求する。
3. **ノイズを排除する**:
   - UI は簡素であることを理想とする。
   - 過度な装飾や色使い、過剰なログ・警告を避ける。
4. **観察を加速する**:
   - データの比較、結合、基礎統計（NULL率, min/max等）へのアクセスを容易にする。

## Decision Rule (意思決定基準)
新しい機能や修正を検討する際は、以下のいずれかを満たすか自問すること：
- 見やすさが向上するか？
- 思考を止めないか？
- 気づきを増やすか？
- 軽さを損なわないか？

**これらに反する提案や実装は行わない。**

## Workflow Mandates (運用ルール)
1. **計画の運用**:
   - 常に `PLAN.md` を最新に保ち、未完了のタスクを管理する。
   - **完了したタスクは `PLAN.md` から削除し、`HISTORY.md` へ移動して記録すること。**
2. **教訓の蓄積**:
   - 開発中に得た知見や回避した問題は `LESSONS.md` に追記する。

## Non-Goals & Constraints
- フル BI 機能や過度な GUI 化は目指さない。
- AI 機能はあくまで「補助」であり、ユーザーの観察と判断を主役とする。
- 異種 DB の統合は、DB 側での直接結合ではなく「ローカルへの持ち寄り」戦略をとる。
- DB 方言の完全な吸収に固執し、複雑性を増大させない。
